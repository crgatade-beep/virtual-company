# ops/sop_design.md
Skill ID: ops-sop-01
Category: Operations
Agent: O-01, MGR-OPS
Difficulty: Intermediate

## Definition
Creating Standard Operating Procedures — repeatable, measurable, auditable workflows that ensure consistent output and enable scaling.

## SOP Structure
```
1. PURPOSE (1-2 sentences why this exists)
2. SCOPE (who uses it, when)
3. DEFINITIONS (key terms)
4. STEP-BY-STEP PROCEDURE (numbered, no ambiguity)
5. QUALITY GATES (checkpoints, not just at end)
6. ESCALATION PATHS (who to call when things break)
7. METRICS (how we measure success)
8. REVISION HISTORY (date, author, change description)
```

## Writing Rules
- Max 10 steps per SOP
- Use active voice: "Click Submit", not "The button should be clicked"
- Include screenshots/diagrams for UI steps
- Write for a new hire with zero context
- No jargon without definition

## Maintenance
- Review quarterly
- Trigger update when: tool changes, error rate > 2%, team feedback
- Version control all SOPs
- Publish to shared knowledge base

## KPIs
- SOPs documented: All critical processes
- Team compliance: 90%+
- Error rate from process failures: < 2%
