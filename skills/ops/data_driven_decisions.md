Category: Operations

Difficulty: Intermediate

## Definition
Making operational and strategic decisions based on data evidence rather than intuition, opinion, or tradition.

## Decision Framework
1. **Define question**: What exactly are we deciding?
2. **Identify metric**: Single metric that answers the question
3. **Collect data**: Current + historical + benchmarks
4. **Analyze**: Trend, comparison, statistical significance
5. **Decide**: What the data says vs what stakeholders want
6. **Validate**: A/B test if possible
7. **Review**: Did prediction match outcome?

## Must-Know Metrics by Function
- **Sales**: Pipeline, win rate, LTV/CAC, churn
- **Product**: DAU/MAU, NPS, feature adoption
- **Tech**: Uptime, MTTR, deploy freq, error budget
- **Finance**: Burn rate, gross margin, runway, NPV

## Common Biases to Avoid
- **Survivorship bias**: Only looking at successes
- **Confirmation bias**: Seeing what supports our hypothesis
- **Anchoring**: Overweighting first data point
- **Sunk cost**: Continuing because of past investment

## KPIs
- Decisions backed by data: 80%+
- Data availability for key decisions: 95%+
- Decision accuracy: 70%+
