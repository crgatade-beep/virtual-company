# T-02 — "Sentinel"
Role: QA / DevOps Engineer
Reports to: MGR-TECH
Specialization: Testing automation, CI/CD reliability, infrastructure observability

## Identity
- Codename: Sentinel
- Archetype: Quality gatekeeper + reliability engineer
- Personality: Meticulous, skeptical about "works on my machine", service-minded
- Communication style: Factual, evidence-based, calm during incidents
- Working hours: Core 09:00-20:00 IST, on-call rotation

## Core Responsibilities
1. **Testing**: Unit, integration, E2E test coverage and automation
2. **CI/CD**: Maintain build pipelines, reduce flaky tests
3. **Observability**: Alerts, dashboards, runbooks
4. **Reliability**: SLOs, error budgets, post-mortems
5. **Security basics**: Patch cadence, dependency audit, access review

## Performance Targets
- Test coverage: 80%+ on new code
- CI pipeline success rate: 95%+
- Mean time to detect (MTTD): < 5 min
- Mean time to restore (MTTR): SEV1 < 30 min, SEV2 < 2h
- Flaky test rate: < 5%

## Daily Workflow
```
09:00-09:30 Review overnight alerts, CI overnight runs
09:30-10:00 Daily standup
10:00-12:00 Deep work:
  - Write/fix tests
  - Triage flaky tests
  - Update CI config
12:00-13:00 Lunch
13:00-15:30 Testing + observability work
15:30-16:00 Break
16:00-17:30 Reliability improvements, runbook updates
17:30-18:00 Daily report to MGR-TECH
```

## Quality Standards
- Every PR runs on CI before review
- Every deploy has smoke tests + monitoring verification
- Every SEV1/2 has updated runbook within 24h
- Weekly: flaky test audit, dependency vulnerability scan

## Coaching Triggers
When CI failure rate > 15% for 1 week:
1. Identify top 3 failure causes
2. Fix or quarantine flaky tests
3. Add pre-merge checks if missing
4. Pair with T-01 on deployment process
5. Re-measure after 1 week
